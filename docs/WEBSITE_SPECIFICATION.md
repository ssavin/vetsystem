# Техническое задание: Корпоративный сайт VetSystemAI

## 1. Общая информация

### 1.1 Наименование проекта
**VetSystemAI** — корпоративный маркетинговый сайт с интеграцией личного кабинета клиентов ветеринарных клиник.

### 1.2 Домен
`vetsystemai.ru`

### 1.3 Цели сайта
1. **Привлечение новых клиник** — презентация возможностей платформы
2. **Конверсия посетителей** — регистрация на демо, заявки на подключение
3. **Вход для клиентов клиник** — личный кабинет владельцев животных
4. **SEO-продвижение** — органический трафик по ключевым запросам
5. **Поддержка существующих клиентов** — база знаний, документация

### 1.4 Целевая аудитория
| Сегмент | Описание | Цель визита |
|---------|----------|-------------|
| Руководители клиник | Принимают решение о покупке | Оценить возможности, запросить демо |
| Администраторы клиник | Ищут замену текущей системе | Сравнить функционал, цены |
| IT-специалисты | Оценивают техническую сторону | Интеграции, API, безопасность |
| Владельцы животных | Клиенты клиник-партнёров | Вход в личный кабинет |

## 4. Технические требования

### 4.1 Стек технологий

| Компонент | Технология |
|-----------|------------|
| Frontend | Next.js 14+ (App Router) |
| Стилизация | Tailwind CSS + Shadcn/ui |
| Анимации | Framer Motion |
| CMS для блога | Contentful / Strapi / MDX |
| Формы | React Hook Form + Zod |
| Аналитика | Яндекс.Метрика + Google Analytics 4 |
| Хостинг | Vercel / собственный сервер |
| CDN | Cloudflare |
| Email | SendGrid / Mailgun |

### 4.2 Интеграция с VetSystem

#### API-эндпоинты для личного кабинета владельцев
```
GET  /api/mobile/auth/send-code     → Отправка SMS-кода
POST /api/mobile/auth/verify        → Верификация кода
GET  /api/mobile/owner/profile      → Профиль владельца
GET  /api/mobile/owner/pets         → Список питомцев
GET  /api/mobile/owner/pets/:id     → Детали питомца
GET  /api/mobile/appointments       → Записи на приём
POST /api/mobile/appointments       → Создание записи
GET  /api/mobile/history            → История визитов
GET  /api/mobile/documents          → Документы
```

#### Авторизация
- JWT-токены
- Refresh-токены
- Хранение в httpOnly cookies

### 4.3 SEO-требования

#### Технические:
- SSR/SSG для всех страниц
- sitemap.xml (автогенерация)
- robots.txt
- Canonical URLs
- Hreflang (ru, en)
- Core Web Vitals: LCP < 2.5s, FID < 100ms, CLS < 0.1

#### Контентные:
- Уникальные title и description для каждой страницы
- H1 только один на странице
- Alt-тексты для изображений
- Structured data (Organization, Product, FAQ, Article)

### 4.4 Безопасность

- HTTPS обязателен
- CSP (Content Security Policy)
- XSS-защита
- CSRF-токены для форм
- Rate limiting для API
- Защита от спама (reCAPTCHA v3)

### 4.5 Производительность

- Lazy loading изображений
- Оптимизация изображений (WebP, AVIF)
- Code splitting
- Кэширование статики
- Минификация JS/CSS
- Предзагрузка критичных ресурсов

### 4.6 Адаптивность

- Mobile-first подход
- Брейкпоинты: 320px, 640px, 768px, 1024px, 1280px
- Тестирование на iOS Safari, Android Chrome
- PWA-манифест для установки на домашний экран
